纯前端SVG绘制工具 #

 工具使用 SVG 实现画布与绘制，因为SVG有矢量效果，可以在绘制完成后导出作为 .svg 使用。

 项目首先采用 VUE3+TS+VITE 开发和打包，Node版本为 v16.20.2 。

实现方案概述 #

1. 总体展示实现 #

定义图元的通用数据结构：interface ElementType 定义字段type来区分元素类型，如：圆形、矩形、直线等。定义用于存储所有元素的数组：const doneElements: ElementType[]。绘制基础图形元素的时候，根据元素类型，创建/修改对应的对象：const currentElement:ElementType，并将当前的数据对象currentElement加入/更新到数组doneElements中。其中，类型定义参照svg基础图形：元素数据结构。通过v-for循环的方式，在SVG画布中，绘制doneElements中的所有元素。示例如下代码片段：

<div class=”svg-box”>
<svg 
  xmlns="http://www.w3.org/2000/svg"
  xmlns:xlink="http://www.w3.org/1999/xlink"
  id="svg-id"
>
    <g  v-for=”item in doneElements” :key=”id”>
        <line 
        v-if=”item.type == ‘line’” 
        :x1=”item.x1” :y1=”item.y1” 
        :x2=”item.x2” :y2=”item.x2”
        :stroke=”item.stroke”
        :stroke-width=”item.strokeWidth”
        :transform=”rotate(item.angle, item.x1+item.width/2, item.y1+item.height/2)  scale(item.scaleX, item.scaleY)”
        </line>
    ···其他类型的svg元素
    </g>
</svg>
</div>

基本上其实就是基于不同类型的svg标签，基于类型进行循环插入到g标签内，所有的g标签都置于svg标签内，最终将svg标签渲染到页面，即可展示出所有页面上的内容；

2. 数据插入实现 #

我们先预置一部分简单图形，来用作绘制复杂图形，在左侧的菜单结构内生成一个Json数组，对简单图形进行遍历展示，这样即可再左侧生成简单图形的svg标签列表，给列表内的item节点添加draw拖拽事件，监听他的拖出过程，并将其记录到全局的create_json中，然后我们在右侧画布区域主体上增加draw事件，监听拖入放置，根据鼠标最终落点松开时的位置获取到x，y值，将其记录，便于作为该拖入的简单图形的中心点x,y坐标使用，并且会根据中心点做拖入的简单图形的宽高计算，进行插入到全局的图形数组 ElementType[]内，做到实时更新全局数据，同步渲染到svg画布。
示例如下：

 // 设置当前画布上的所有元素 -- 图元编辑页
    setDoneJson = (done_json: IDoneJson[] | IDoneJson) =>{
      if (isOfType(done_json, 'id')) {
        this.done_json.push(done_json);
        nextTick(() => {
          setSvgActualInfo(done_json);
        });
      } else {
        this.done_json = objectDeepClone<IDoneJson[]>(done_json);
      }
    }

其中 setSvgActualInfo方法即是传入当前插入的简单图形的json对象进行宽高，xy等参数计算；
示例如下：

/**
 * @param done_json 传入的解析json 为新插入的简单图形json对象
 * 该方法为注册给出的json内的所有元素的部分基础属性
 * 进入页面时，如果元素未设置，则赋予的actual_bound对象内的x,y,width,height皆为0，在此处对元素做计算处理
 */
export const setSvgActualInfo = (done_json: IDoneJson) => {
  const queryBbox = document.querySelector(`#${done_json.id}`);
  const rectBBox = document.querySelector(`#rect${done_json.id}`);
  if (queryBbox) {
    let x = 0,
      y = 0,
      width = 0,
      height = 0;
    // 补充必要缺失属性
     const BBox = (queryBbox as SVGGraphicsElement).getBBox();
      x = BBox.x;
      y = BBox.y;
      width = BBox.width;
      height = BBox.height;
    if (
      rectBBox &&
      rectBBox.getAttribute('x') === '0' &&
      rectBBox.getAttribute('y') === '0' &&
      rectBBox.getAttribute('width') === '0' &&
      rectBBox.getAttribute('height') === '0'
    ) {
      rectBBox.setAttribute('x', x.toString());
      rectBBox.setAttribute('y', y.toString());
      rectBBox.setAttribute('width', width.toString());
      rectBBox.setAttribute('height', height.toString());
    }
    //实际大小和坐标理论上不会变 但是如果子组件设置了100% 还是会变 所以要做下判断
    if (
      done_json.actual_bound.x === 0 &&
      done_json.actual_bound.y === 0 &&
      done_json.actual_bound.width === 0 &&
      done_json.actual_bound.height === 0
    ) {
      //如果该对象内的值皆为0 则对齐进行计算属性的赋值
      done_json.actual_bound = { x, y, width, height };
    }
  }
};

我们可以很清晰的看到当我们进行插入图形的时候，进行了两步操作，首先对其进行宽高、xy计算，其次将其插入全局数组进行保存；
那么就有人产生疑问，为什么必须要计算他的宽高才能进行放置，正常的简单svg图形，例如rect标签都是有自身的宽高的，难道不足以展示到页面上吗？
其实不然，正常插入的简单图形，矩形rect、圆形circle等等，都是有自身的坐标和半径等参数作为展示标准的，但是在我们的数组内并不统一，就会导致我们在导出svg文件时，无法实际的计算页面上的图形大小，无法准确的裁剪出我们需要的大小，所以对每个图形的宽高计算就显得尤为重要。不仅在导出时非常需要该参数，在我们后续加入的拖拽放大缩小等操作中，更需要这些计算参数作为基底来进行实现。
接下来我们看一下最终的结果产出---导出svg文件；在导出中，我们需要获取svg的dom，剔除不需要的个别内容，使用最终的xy坐标和四个方位最极点的图形，对其进行宽高、xy计算，获得整张svg的裁剪范围，并使用viewBox进行范围展示；
代码示例如下：

  //保存为SVG图片
  const onSave = (data_model: IDataDoneModel) => {
    // 判断画布元素不能为空
    if (data_model.done_json.length === 0) {
      ElMessage.warning('画布中元素为空，无需导出SVG文件！');
      return;
    }
    let svgElement: any = document.getElementById('svg-id')?.outerHTML;
    let elements = '';
    //去掉网格辅助线等相关
    const gridStart = svgElement.indexOf('<defs');
    const gridEnd = svgElement.indexOf('fill="url(#pattern_grid)"');
    svgElement =
      svgElement.substring(0, gridStart) +
      svgElement.substring(gridEnd + 'fill="url(#pattern_grid)"'.length + 8);
    //设置背景色为透明
    svgElement =
      svgElement.substring(0, svgElement.indexOf('style=')) +
      'style="background-color: transparent"' +
      svgElement.substring(svgElement.indexOf('>'));
    const inputStart = svgElement.substring(
      svgElement.indexOf('class="imp_btn">'),
      svgElement.indexOf('class="aa">')
    );
    //设置viewbox，否则超出屏幕的元素，在导出文件中无法展示
    const offsetX = data_model.layout_center.x + data_model.config.position_center.x;
    const offsetY = data_model.layout_center.y + data_model.config.position_center.y;
    const xMinList: number[] = [];
    const xMaxList: number[] = [];
    const yMinList: number[] = [];
    const yMaxList: number[] = [];
    // 获得四个角的极点图形坐标数组
    data_model.done_json.forEach((node) => {
      xMinList.push(node.x - (node.actual_bound.width * node.scale_x) / 2);
      xMaxList.push(node.x + (node.actual_bound.width * node.scale_x) / 2);
      yMinList.push(node.y - (node.actual_bound.height * node.scale_y) / 2);
      yMaxList.push(node.y + (node.actual_bound.height * node.scale_y) / 2);
    });
    // 对比产出四个角的极点图形坐标
    const startX: number = parseFloat((Math.min(...xMinList) + offsetX).toFixed(2));
    const startY: number = parseFloat((Math.min(...yMinList) + offsetY).toFixed(2));
    const endX: number = parseFloat((Math.max(...xMaxList) + offsetX + 100).toFixed(2));
    const endY: number = parseFloat((Math.max(...yMaxList) + offsetY + 100).toFixed(2));
     // 给了一定的偏移量，避免svg产出的过于小，固定一下最小尺寸 
    let svgWidth = Math.abs(endX - startX) < 600 ? 600 : Math.abs(endX - startX);
    let svgHeight = Math.abs(endY - startY) < 400 ? 400 : Math.abs(endY - startY);
    // 产出svg文件的 viewBox 控制展示窗口
    svgElement =
      svgElement.substring(0, 4) +
      ` viewBox="${startX} ${startY} ${svgWidth} ${svgHeight}" ` +
      svgElement.substring(4);
    //去掉字符串中的换行
    svgElement = svgElement.replace(/(\r\n|\n|\r)/gm, '');
    svgElement = vkbeautify.xml(svgElement);
    const blob = new Blob([svgElement], { type: 'text/plain;charset=utf-8' });
    // 进行导出
    FileSaver.saveAs(
      blob,
      `${globalStore.done_json_message.name}-${moment(new Date()).format(
        'YYYY-MM-DD HH:mm:ss'
      )}.svg`
    );
  };

以上，我们就完成了一份简单的svg绘制工具的前端代码制作，目前我们未使用到后端，原因是我们是即时绘制并导出为svg文件的，但在绘制过程中，避免误触操作，我们可以增加一个每三分钟的覆盖localStorage数据json，如果在绘制中出错，也可以使用它回到原始版本，也可以在误触刷新页面后及时恢复数据，防止做无用功！

 //设置定时器 以每三分钟提交一次localstorage内容 防止页面刷新导致的丢失
